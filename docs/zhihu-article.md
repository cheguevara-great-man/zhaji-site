# 我把自己的知乎内容迁移成了一个独立网站

这两天做了一个小项目：把我在知乎上写过的文章、回答和想法，迁移到一个自己管理的网站里。

项目地址：

https://github.com/cheguevara-great-man/zhaji-site

网站目前已经能在服务器上运行，支持公网访问。它不是一个很复杂的产品，但对我来说意义比较明确：把自己长期写下来的东西，从平台的信息流里拎出来，放到一个更稳定、更可控的地方。

## 为什么要做这个

人在网上写东西，很多时候会有一种错觉：内容是自己的。

但实际上，内容往往被平台的结构重新安排了。文章、回答、想法混在不同入口里，被推荐机制、搜索排序、账号状态和平台规则支配。写的时候是自己的表达，过一段时间再看，就变成了平台数据库里的某个条目。

我想要一个更简单的地方：

- 文章、回答、想法都放在一起；
- 可以按类型归档；
- 可以保留原文链接；
- 可以自己决定页面长什么样；
- 可以让别人访问、注册、评论；
- 不依赖某个平台的展示逻辑。

所以做了这个网站。

我给它起的名字叫“札记”。首页上写的是：

> 思想是人在时间里存在的痕迹。  
> 文章、回答、想法、小说和随笔都放在这里。

这基本就是这个项目的定位。

## 它现在能做什么

这个代码库目前实现了一个完整的个人内容站。

前台主要有两个部分：

- 首页：按时间展示最近内容，支持滚动加载；
- 目录页：按全部、文章、回答、想法分类浏览。

每一篇内容都有独立页面。页面里保留知乎原始创建时间、原文链接和正文内容。列表页会展示摘要，详情页则直接进入正文，不再重复摘要。

后台和账号系统也做了：

- 邮箱注册；
- 邮箱验证；
- 登录；
- 找回密码；
- 登录后评论；
- 管理员可以管理内容。

数据暂时用 JSON 文件保存，没有上复杂数据库。这是有意的：个人站点不需要一开始就上很重的架构。JSON 对个人规模的内容已经够用，也便于备份、迁移和检查。

## 最重要的部分：自动同步知乎

这个项目最关键的功能，是自动从知乎同步内容。

现在同步逻辑不是简单粗暴地每天全量抓一遍正文，而是做成了两段式：

第一步，只读取知乎列表里的元数据：

- 原文地址；
- 标题；
- 知乎创建时间；
- 知乎修改时间；
- 内容类型。

第二步，把这些元数据和本地数据库比较。

如果 `sourceUpdatedAt` 没变，就认为内容没变，直接跳过，不再读取正文，也不下载图片，更不写数据库。

只有在两种情况下才会继续抓正文：

- 本地没有这条内容，也就是新增；
- 知乎上的修改时间变了，也就是内容被编辑过。

这样做的好处很明显：省时间，也省请求量。

我现在有几百条内容，如果每天都把所有正文重新抓一遍，既慢，也容易触发平台风控。现在只读列表元数据，没变化的内容直接跳过，日常同步会轻很多。

文章和回答比较适合这种两段式同步。想法稍微特殊一点，因为知乎的想法列表接口本身就会返回内容，所以没法完全避免读取想法正文。但即便如此，脚本也会根据修改时间避免重复写库和重复处理图片。

## 时间字段怎么处理

我不想用“导入到本站的时间”来冒充内容时间。

所以站里保存了两个知乎来源时间：

- `sourceCreatedAt`：知乎内容创建时间；
- `sourceUpdatedAt`：知乎内容修改时间。

页面展示和排序主要看 `sourceCreatedAt`。

同步判断主要看 `sourceUpdatedAt`。

这样更符合内容本身的历史，而不是本站导入数据库的历史。

## 部署方式

这个网站已经部署到一台 Ubuntu 服务器上。

大致结构是：

- Node.js 运行网站服务；
- Nginx 反向代理到 80 端口；
- systemd 管理网站进程；
- systemd timer 每天自动同步知乎；
- 服务器本地保存知乎登录态；
- 每天自动备份数据文件。

知乎登录不是用账号密码硬登，而是在服务器上启动 Playwright/Chromium，生成二维码截图，用手机知乎扫码一次。扫码后，服务器自己的浏览器 profile 保存登录态。之后定时任务就可以在服务器上自动同步。

这个登录态会失效。失效后重新扫码即可。

## 为什么公开这个代码库

这个项目当然不是什么大型系统，也不是通用 CMS。

但我觉得它对一类人可能有用：

- 长期在知乎写东西的人；
- 想把内容从平台迁移出来的人；
- 想要一个自己的个人内容归档站的人；
- 不想一开始就折腾复杂数据库、后台系统和部署流程的人；
- 想看一个从抓取、导入、展示、评论到部署都能跑通的小项目的人。

它不是完美的。

比如现在还没有特别复杂的权限系统，也没有富文本编辑器，没有全文搜索，没有图片管理后台，也没有多用户作者体系。但作为一个个人站点的起点，它已经够用了。

更重要的是，它把一件事跑通了：把分散在知乎里的内容，整理成一个自己可控的网站。

## 后面可能继续做什么

后面可能会继续加这些功能：

- 全文搜索；
- 标签系统；
- 更好的评论管理；
- RSS；
- 后台同步日志页面；
- 删除内容时增加确认队列；
- 更好的移动端阅读体验；
- 从其他平台导入内容。

但目前这个版本已经能解决最核心的问题：内容能迁移，网站能访问，知乎能自动同步，评论和账号系统能用。

如果你也想把自己的知乎内容整理成一个独立网站，可以参考这个项目：

https://github.com/cheguevara-great-man/zhaji-site

它不一定适合所有人，但至少说明了一件事：个人写作者不一定只能被平台的信息流保存。你可以把自己的文字重新收回来，按自己的方式保存、展示和讨论。
